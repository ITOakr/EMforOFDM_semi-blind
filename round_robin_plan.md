# 実装計画: AIC全パターン総当たりによる正解パス探索能力の検証 (8パス制限版)

## 1. 概要
現在のAICモデル選択は、電力順にパスを並べて上位から順に検討する「簡略版（Wrapper法）」です。本モード（Mode 41）では、探索空間を最初の **8パス** に限定し、その全 $2^8 = 256$ 通りのパスマスク構成をすべて計算し、AICが最小となるマスクを特定します。
これにより、AICという指標自体が真のモデルを識別する能力を、探索アルゴリズムの制約なしに評価します。

※OpenMP（並列化）は使用せず、シングルスレッドで実装します。

## 2. 評価指標
*   **完全一致正答率 (Exact Match Accuracy)**:
    総当たりで選ばれたパスマスクが、真のパスマスク（`params_.pathMask` の最初の8要素）と全ビット完全に一致した試行の割合。
*   **Eb/N0 スイープ**:
    ノイズレベルを変化させ、正答率がどのように推移するかを観測。

## 3. 実装ステップ（必須チェックリスト）

### ステップ 1: `Transceiver.h` への総当たり探索メソッドの追加
**【厳守事項】AICの計算式は Mode 12 (`set_initial_params_by_pilot`) と完全に一致させること。**

*   **メソッド名**: `std::vector<int> findBestMaskByExhaustiveAIC_8paths()`
*   **処理ロジック**:
    1.  `min_aic` を無限大で初期化。
    2.  $i = 1$ から $255$ ($2^8-1$) までループ。
    3.  整数 $i$ の下位8ビットをパスマスクに変換。
    4.  そのマスクを用いて LS 推定を行い、残差電力を計算。
    5.  **AIC算出 (Mode 12 完全準拠)**:
        *   `double beta = (double)params_.K_ / residual;`
        *   `double logL = params_.K_ * std::log(beta) - params_.K_ * std::log(M_PI) - params_.K_;`
        *   `double aic = -2.0 * (logL - 2.0 * Q_active);`
    6.  最小の AIC を与えたマスクを保持し、最後に返す。

### ステップ 2: `simulator.h` へのシミュレーションメソッドの実装
**【厳守事項】チャネル生成は「最初の8パス」のみが有効になるように制限すること。**

*   **メソッド名**: `double getExhaustiveAICAccuracy_8paths_Simulation()`
*   **処理ロジック**:
    1.  正解カウントをリセット。
    2.  試行ループ内で以下を実行:
        *   **チャネル生成の制限**: `params_.pathMask` をランダムに生成する際、インデックス 0〜7 の範囲でのみパスを配置し、8〜15 は常に 0 に固定する（検証空間と生成空間を一致させるため）。
        *   `transceiver_.setX_()`, `channel_.generateFrequencyResponse(fd_Ts_)`, `transceiver_.setY_()` を実行。
        *   `transceiver_.findBestMaskByExhaustiveAIC_8paths()` を呼び出す。
        *   返されたマスクと、その試行の真の `pathMask` (上位8bit) を比較し、完全一致すればカウントアップ。
    3.  正答率（カウント / 試行回数）を返す。

### ステップ 3: `main.cpp` への組み込み
Mode 41 として実装し、結果をCSV出力します。

*   **ファイル名**: `..._ExhaustiveAIC8_Accuracy_EbN0.csv`
*   **スイープ**: Eb/N0 をスイープ。

## 4. 特記事項
*   **計算負荷**: シングルスレッドで 256 パターン/試行。試行回数 1,000〜10,000 回程度を想定。
*   **整合性**: ステップ1のAIC式と、ステップ2の生成範囲制限が噛み合っていることが、本検証の公平性の担保となります。

## 5. 理論的期待
*   **近似の限界の特定**: Mode 12（電力順）で正解できなかった試行が、Mode 41（総当たり）で正解できるようになるかを確認します。もしどちらも間違える場合は、AICの数式そのものがレイリーフェージング下での疎なパス特定に限界があることを示唆します。
